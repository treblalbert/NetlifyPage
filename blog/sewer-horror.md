# Sewer Horror

Hello and happy Sunday!

This past week I remembered that last year I paid for a publisher account so I could release games on the Google Play Store. Since I am currently very unemployed, I decided to set myself a challenge: make a decent mobile game and properly learn how to publish on Android, including ads, store setup, and everything that comes with it.

And so I did.

The game is almost finished. In the end, actually making the game took less time than I expected, which is a blessing, because publishing on the Play Store is just as painful as Steam, if not worse. It is genuinely a huge pain, and if my games ever start doing well, I will happily give a percentage to a publisher just to avoid dealing with all of that again.

Now, about the game itself.

![Sewer Horror screenshot 1](blog/Sewer_1.png)

It is called Sewer Horror, and it is exactly what it sounds like. It is a first-person horror game set in procedurally generated sewers. To progress deeper, you need to solve puzzles while exploring the environment. The system is piece-based, which makes it easy to expand. I can keep modeling new sections and adding puzzles, so updating the game should be pretty straightforward.

Of course, it would not be a proper horror game without something chasing you. There are creepy enemies roaming around, and the game includes stealth elements so you can avoid them instead of just running blindly.

![Sewer Horror screenshot 2](blog/Sewer_2.png)

I will make another post when it is released.

Funny thing is, I originally wanted to make a quick and easy cash grab since, as I said, I am broke right now. But I felt bad making something lazy, so I pushed it further. And honestly, the game turned out to be genuinely fun and kind of addictive. During playtesting, I kept playing way longer than I expected.
