🛠️ How This Site Was Made

A simple guide to how we built and maintain this project site

📁 GitHub - The Code Storage

Think of GitHub as a cloud storage for code. Just like Google Drive stores your documents, GitHub stores all the website files (HTML, images, etc.).

• All our code lives in a GitHub repository
• It keeps track of every change we make
• Multiple people can work on it together

🌐 Netlify - The Website Host

Netlify is like a TV broadcaster for websites. It takes our code from GitHub and broadcasts it to the internet so everyone can see it.

• Automatically publishes our site when we update code
• Makes it available at our web address
• Free for small projects like ours!

⚡ The Magic One-Liner

We use a simple command that does three things at once:

git add . && git commit -m "Auto commit" && git push

What it does:
1. git add . - Grabs all changed files
2. git commit - Packages them with a note
3. git push - Sends them to GitHub

🔄 The Automatic Workflow

  1. We edit files on our computer
  2. Run the magic one-liner command
  3. Files get uploaded to GitHub
  4. Netlify sees the change and updates the live site
  5. Changes appear online in ~30 seconds!

📝 The Notes System

This notes feature uses Supabase - think of it as a Google Sheets that websites can talk to.

• Notes are stored in the cloud
• Anyone on the team can add/edit notes
• Changes sync instantly for everyone
• No need to update code for new notes!

✨ The Beauty of Automation

With this setup, updating the website is as easy as saving a Word document. No complex server management, no FTP uploads, just save and push!
